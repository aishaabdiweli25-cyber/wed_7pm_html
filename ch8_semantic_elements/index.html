Build with semantic HTML

This small single-file web app demonstrates how to structure a page using semantic elements: <header>, <nav>, <main>, <article>, <section>, <aside>, and <footer>. No JavaScript — just HTML & CSS.

Key features

Examples of semantic markup that improve accessibility and maintainability.

  • Semantic elements: header, nav, main, article, section, aside, footer, figure, figcaption.
  • Accessible labels: aria-labels and landmarks help screen readers.
  • Responsive layout: grid + media queries for small screens.
  • No JavaScript: works even when scripts are disabled.
